## 2. DEFINICJE I PODSTAWOWE INFORMACJE

### 2.1 Czym są cookies?
**Cookies** (ciasteczka) to małe pliki tekstowe zapisywane na Twoim urządzeniu (komputer, tablet, smartfon) przez przeglądarkę internetową podczas odwiedzania stron internetowych. Zawierają informacje, które są odczytywane przez serwer WWW przy kolejnych wizytach.

### 2.2 Technologie podobne do cookies
Oprócz tradycyjnych cookies stosujemy również:
- **Local Storage** - lokalne przechowywanie danych w przeglądarce
- **Session Storage** - dane sesji usuwane po zamknięciu przeglądarki
- **Web Beacons** - niewidoczne obrazki do śledzenia aktywności
- **Fingerprinting** - identyfikacja na podstawie konfiguracji urządzenia
- **ETags** - nagłówki HTTP do śledzenia zasobów

### 2.3 Rodzaje cookies według żywotności
1. **Cookies sesyjne** - usuwane po zamknięciu przeglądarki
2. **Cookies trwałe** - pozostają na urządzeniu przez określony czas
3. **Cookies własne** - ustawiane przez domenę cenysync.pl
4. **Cookies stron trzecich** - ustawiane przez zewnętrzne serwisy

### 2.4 Cele stosowania cookies
- **Funkcjonalność** - zapewnienie prawidłowego działania serwisu
- **Bezpieczeństwo** - ochrona przed atakami i nadużyciami  
- **Personalizacja** - dostosowanie treści do preferencji użytkownika
- **Analityka** - pomiar ruchu i optymalizacja wydajności
- **Marketing** - personalizacja reklam i remarketing

## 8. BEZPIECZEŃSTWO I OCHRONA COOKIES

### 8.1 Środki techniczne bezpieczeństwa

#### Flagi zabezpieczające
- **Secure** - cookies przesyłane tylko przez HTTPS
- **HttpOnly** - niedostępne dla JavaScript (protection przed XSS)
- **SameSite** - ochrona przed atakami CSRF
  - `Strict` - tylko same-site requests
  - `Lax` - ograniczone cross-site requests  
  - `None` - wymaga Secure flag

#### Szyfrowanie i hashowanie
```javascript
// Przykład zabezpieczenia cookie
Set-Cookie: cenysync_session=encrypted_value; 
           Secure; HttpOnly; SameSite=Strict; 
           Max-Age=3600; Path=/; Domain=.cenysync.pl
```

#### Walidacja po stronie serwera
- **Token validation** - weryfikacja autentyczności cookies
- **Session management** - kontrola aktywnych sesji
- **Anomaly detection** - wykrywanie nietypowych wzorców
- **Rate limiting** - ochrona przed atakami brute-force

### 8.2 Monitoring i audyty
**Regularne procedury bezpieczeństwa:**

1. **Quarterly reviews** - przegląd wszystkich aktywnych cookies
2. **Security scans** - automatyczne testy podatności
3. **Third-party audits** - zewnętrzne audyty bezpieczeństwa  
4. **Compliance checks** - zgodność z najnowszymi wytycznymi

### 8.3 Incydenty bezpieczeństwa
**Procedura w przypadku naruszenia cookies:**

1. **Natychmiastowa reakcja** - wylogowanie wszystkich użytkowników

3. **Analiza** - identyfikacja przyczyny i skali naruszenia

- **Aktualizacja profilu** - zmiana preferencji i ustawień

#### Prawo do usunięcia (art. 17)
- **Usunięcie cookies** - żądanie skasowania wszystkich lub określonych
- **Prawo do bycia zapomnianym** - kompletne usunięcie profilu

#### Prawo do ograniczenia (art. 18)
- **Zawieszenie przetwarzania** - tymczasowe wstrzymanie cookies
- **Ograniczenie celów** - używanie tylko do określonych funkcji

#### Prawo do przenoszenia (art. 20)
- **Eksport preferencji** - pobranie ustawień w formacie JSON/CSV
- **Transfer do innego serwisu** - migracja danych cookies

#### Prawo sprzeciwu (art. 21)
- **Sprzeciw wobec profilowania** - zaprzestanie analizy zachowań

### 12.2 FAQ - Najczęściej zadawane pytania

#### Q: Czy mogę korzystać z CenySync bez cookies?
A: Tak, ale z ograniczeniami. Cookies niezbędne są wymagane do podstawowego działania (logowanie, bezpieczeństwo). Bez cookies funkcjonalnych i analitycznych tracisz personalizację i możliwość optymalizacji doświadczenia.

#### Q: Jak często aktualizujecie listę cookies?
A: Pełny audit przeprowadzamy kwartalnie. Nowe cookies są dodawane do polityki w ciągu 7 dni od implementacji. Użytkownicy są informowani o istotnych zmianach.

#### Q: Czy cookies wpływają na wydajność platformy?
A: Minimalizujemy wpływ przez optymalizację rozmiaru cookies, lazy loading trackerów stron trzecich i cache'owanie. Średni wpływ na czas ładowania <50ms.

#### Q: Jak długo przechowujecie cookies?
A: Od sesyjnych (usuwalnych po zamknięciu przeglądarki) do maksymalnie 2 lat dla niektórych cookies analitycznych. Szczegóły w tabelach powyżej.

#### Q: Czy przekazujecie cookies podmiotom trzecim?
A: Bezpośrednio nie. Jednak cookies stron trzecich (Google, Facebook) mogą być przekazywane ich systemom zgodnie z ich politykami prywatności.

// Przykład implementacji cookie consent managera
const CookieConsent = {
  categories: {
    necessary: { required: true, active: true },
    functional: { required: false, active: false },
    analytics: { required: false, active: false },
    marketing: { required: false, active: false }
  },
  
  updateConsent: function(category, status) {
    this.categories[category].active = status;
    this.enforceCookiePolicy();
    this.savePreferences();
  },
  
  enforceCookiePolicy: function() {
    if (!this.categories.analytics.active) {
      this.removeGoogleAnalytics();
    }
    if (!this.categories.marketing.active) {
      this.removeMarketingPixels();
    }
  }
};
